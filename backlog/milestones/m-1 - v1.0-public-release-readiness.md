# v1.0 — public release readiness

First public OpenWhisper release. Scope is **polish + open-source readiness**, not new product surface. The dictation engine, Pill HUD, Settings, Mac+Win parity, fullscreen behavior, audio ducking, and launch-at-login are all already shipped on `main` — v1.0 is about making the project mergeable, contributable, and brand-clean for an external audience.

## Pillars

1. **Library API + headless CLI.** Lift the `core/` crate's public Rust API to be ergonomic for an external consumer (record / transcribe / enumerate / cleanup / diagnostics / config). Add a thin `cli/` crate that wraps the same API. The Tauri shell calls the same library functions — by construction, **CLI feature surface = UI feature surface = library surface**. Reference: Tailscale's `cmd/tailscale` + `cmd/tailscaled` over shared internal packages.
2. **Diagnostics.** Crash reports (TASK-78) so first-touch users can tell us what broke without a screen-share.
3. **OSS scaffold.** `SECURITY.md`, `CODE_OF_CONDUCT.md`, `CHANGELOG.md`, `ISSUE_TEMPLATE/`, `dependabot.yml`, `CODEOWNERS`, and a PR-gate CI workflow (clippy + cargo test + Playwright). Brings the repo up to the bar contributors expect.
4. **README + BUILD.md polish.** Rewrite README around the v1 story; consolidate INSTALL.md + dev-loop notes into one `BUILD.md`; externalize the architecture rule from the `openwhisper-orchestration-in-rust` skill into a public contributor doc.
5. **Rename — last step.** Disambiguate from `OpenWhispr/openwhispr` as a final sweep before going public. Touches repo name, crate names, app bundle identifiers, settings paths, README, NOTICE, in-app strings, GitHub org. Everything in pillars 1-4 ships under the current `OpenWhisper` name, then rename does mass find-and-replace + repo rename + bundle path moves on the way out the door.

GitHub Actions cost note: free for public repos (no minute caps). Until the rename + public-flip ships, CI runs on the private repo eat the 2000-min/month free tier (macOS counts 10×, Windows 2×). Practical implication: keep CI minimal until pillar 5; let it breathe afterwards.

## Out of scope

- Linux port.

## Definition of done

- Repo renamed, old name archived/redirected.
- A user who lands on the GitHub repo cold can: read the README, file a bug via issue template, build from source per a single doc, and run the headless CLI to repro a transcription.
- A panic in production lands in the user's `~/Library/Application Support/<new-name>/crashes/` (Mac) and `%APPDATA%\<new-name>\crashes\` (Win), surfaces in Diagnostics, and can be opt-in uploaded.
- `pnpm test:ui` and `cargo test --all-features` run on every PR via GitHub Actions.
- Tagged `v1.0.0` build is signed, notarized, and downloadable from GitHub Releases (Mac DMG + Win MSI).
